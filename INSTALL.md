# Инструкция по установке и запуску

## Требования

- Node.js 18 или выше
- npm или yarn

## Установка

1. Установите зависимости:

```bash
npm install
```

## Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:3000

## Сборка для продакшена

```bash
npm run build
```

Собранные файлы будут в папке `dist/`

## Предпросмотр продакшен сборки

```bash
npm run preview
```

## Важные замечания

### База данных

База данных SQLite инициализируется автоматически при первом запуске приложения. Файлы схемы и начальных данных находятся в папке `database/`:

- `database/schema.sql` - схема базы данных
- `database/seed.sql` - начальные данные (ингредиенты, блюда, рецепты)

База данных работает в браузере через sql.js (WebAssembly), поэтому все данные хранятся в памяти браузера и не сохраняются между сессиями.

### Изображения

Изображения ингредиентов и блюд должны быть размещены в папке `public/images/`:

- `public/images/ingredients/` - изображения ингредиентов
- `public/images/dishes/` - изображения блюд

В текущей версии приложения изображения не обязательны - приложение будет работать и без них.

## Решение проблем

### Ошибка загрузки базы данных

Если приложение не может загрузить базу данных, убедитесь, что:
1. Файлы `database/schema.sql` и `database/seed.sql` существуют
2. Vite правильно настроен для обслуживания статических файлов
3. Нет проблем с CORS (для локальной разработки это не должно быть проблемой)

### Ошибки TypeScript

Если возникают ошибки TypeScript, убедитесь, что все зависимости установлены:

```bash
npm install
```

### Проблемы с Material-UI

Если стили Material-UI не применяются, убедитесь, что установлены все необходимые зависимости:

```bash
npm install @mui/material @mui/icons-material @emotion/react @emotion/styled
```
