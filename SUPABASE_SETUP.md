# Настройка Supabase для What2Eat

## Проблема: Ошибка 404 NOT_FOUND

Если вы видите ошибку `404: NOT_FOUND` от Supabase, это означает, что таблицы не созданы в базе данных.

## Решение: Создание таблиц через SQL Editor

### Шаг 1: Откройте Supabase Dashboard

1. Перейдите на [https://app.supabase.com](https://app.supabase.com)
2. Войдите в свой аккаунт
3. Выберите проект: `siaaptnnchaafzygxugc`

### Шаг 2: Откройте SQL Editor

1. В левом меню найдите **SQL Editor**
2. Нажмите **New Query**

### Шаг 3: Выполните SQL скрипт

1. Откройте файл `scripts/supabase-setup.sql` в этом проекте
2. Скопируйте весь SQL код
3. Вставьте в SQL Editor в Supabase Dashboard
4. Нажмите **Run** (или `Ctrl+Enter`)

### Шаг 4: Проверка

После выполнения скрипта вы должны увидеть сообщение об успешном выполнении. Таблицы будут созданы:

- ✅ `user_profiles` - профили пользователей
- ✅ `ai_recipes` - AI-сгенерированные рецепты
- ✅ `favorite_recipes` - избранные рецепты
- ✅ `user_ingredient_sets` - сохранённые наборы продуктов

## Альтернативный способ: Через скрипт (требует пароль БД)

Если у вас есть пароль базы данных:

```bash
# Установите переменную окружения
$env:DB_PASSWORD="ваш_пароль_базы_данных"

# Запустите скрипт
node scripts/setup-supabase.mjs
```

**Где найти пароль БД:**
1. Supabase Dashboard → Settings → Database
2. Найдите "Database password" или "Connection string"

## После настройки

После создания таблиц ошибка `404 NOT_FOUND` должна исчезнуть, и приложение сможет:
- Сохранять AI-рецепты
- Синхронизировать избранное между устройствами
- Сохранять наборы ингредиентов
